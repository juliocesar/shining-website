Shining

A toolkit for creating in-browser presentations.

Quickstart

Install via rubygems:

$ sudo gem install shining

Create a new presentation:

$ shine mypresentationdir

Intro

Shining helps you creating presentations that you present from a web browser. The presentations that it generates are entirely based in HTML, CSS, with a little help from CSS transforms and transitions, and jQuery.

What makes a presentation?

A presentation is composed of:

  • A configuration file named config.json.
  • Slides, which are plain HTML files in the slides folder.
  • An index file named index.html in the root presentation folder, which is where you start a presentation from.

Configuration

Presentation-wide configuration happens in the config.json file, which as you probably guessed, it's a plain text file containing an object in valid JSON. It's important that it's valid (e.g.: no single quotes) as otherwise your presentation won't even start. Valid attributes are:

  • slides: an array containing the name of the presentation's slides. If you're using the generator to add slides, don't worry about editing it.
  • transitions: a string corresponding to one of the available transition effects, or false disables them altogether.

Slides

Easy way first. Change the directory to the presentation's directory first. Then:

$ shine slide mynewslide

Where "mynewslide" is the name of the slide. This will create a slide in ROOT/slides/mynewslide.html, and a slide script in ROOT/slides/mynewslide.js. It will also automagically edit the config.json file for you, adding a reference to the newly added slide.

You can also create slides manually by adding an HTML file to ROOT/slides, and then adding the file name (minus the extension) to config.json. Though really, just use the damn generator.

Slide templates

Shining supports Haml or Markdown templates for creating slides. Before you can use them though, you need to compile the templates into HTML by running a command.

To generate a Haml template, in a presentation folder type:

$ shine slide myslide haml

Conversely, a Markdown template can be generated by the same command, except use "markdown" instead of "haml".

Once you're done editing the template, compile by:

$ shine compile

Individual slide styles

If a corresponding CSS file exists, it will take effect for as long as a particular slide is loaded. For example, if you have a slide named "intro", having a file intro.css in the slides/ folder will make sure it gets loaded along.

Deploying

If at some stage you wish to publish the presentation on the internet perhaps, or it's best if you vendor all the dependencies first. This will ensure all the necessary libraries required to play your presentation are included. In a gist, you're making your presentation self-contained.

In the presentation's directory, run the command:

$ shine vendor

Then put this folder somewhere accessible via the web, point your browser to it, and you're set.

Effects

Effects are applied on elements by adding CSS classes to them. Effects are based on CSS transitions and/or transforms. What this means is they're likely to work on top notch engines such as Webkit, they may work on Firefox, and most likely will not work with Internet Explorer.

That may or may not change in the near future. For now, stick to using decent web browsers when presenting, and perhaps relying more on content rather than how it looks ;-)
