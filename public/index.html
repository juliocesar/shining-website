Shining

A toolkit for creating in-browser presentations.

Example

A quick example of a presentation using the default theme with fade transitions.

Quickstart

Install via rubygems:

$ sudo gem install shining

Creating a new presentation:

$ shine mypresentationdir

Shining will then copy all necessary files to run a presentation to it's directory.

You can run a presentation locally by opening the index.html file in the presentation's root directory, or run (port number can be whatever number of your choosing):

$ rackup -p 4567

and then point your browser to http://localhost:4567. NOTE: Google Chrome won't allow you to play a presentation locally ever thanks to Javascript security restrictions. So if you're planning on using Chrome, always take the rackup route.

Intro

Shining helps you creating presentations that you can view from a web browser. The presentations that it generates are entirely based in HTML, CSS, with a little help from CSS transforms, transitions, and jQuery.

What makes a presentation?

A presentation is composed of:

  • A configuration file named config.json.
  • Slides, which are plain HTML files in the slides folder.
  • An index file named index.html in the root presentation folder, which is where you start a presentation from.

Configuration

Presentation-wide configuration sits in the config.json file, which is a plain text file containing an object in valid JSON. It's important that it's valid (e.g.: no single quotes) as otherwise your presentation won't even start. Valid attributes are:

slides
an array containing the file names of the presentation's slides. If you're using the generator to add slides, don't worry about editing it.
transitions
a string corresponding to one of the available transition effects, or false disables them altogether.
plugins
a list of plugins that will be used with your presentation. By default, the resize and syntaxhighlighter plugins will get loaded, allowing for automatic font resizing based on screen dimensions and code syntax highlighting, respectively.
title
a title for the presentation.
help
boolean. false disables the growl-like messages that appear on the top right hand corner of the screen.

Slides

Easy way first. Change the directory to the presentation's directory first. Then:

$ shine slide mynewslide

Where "mynewslide" is the name of the slide. This will create a slide in ROOT/slides/mynewslide.html, and a slide script in ROOT/slides/mynewslide.js. It will also automagically edit the config.json file for you, adding a reference to the newly added slide.

You can also create slides manually by adding an HTML file to ROOT/slides, and then adding the file name (minus the extension) to config.json. Though really, just use the damn generator.

Alternative formats

Shining supports Haml or Markdown slides other than HTML.

To generate a Haml template, in a presentation folder type:

$ shine slide myslide haml

Conversely, a Markdown template can be generated a similar command:

$ shine slide myslide markdown

Individual slide styles

If a corresponding CSS file exists, it will take effect for as long as a particular slide is loaded. For example, if you have a slide named "intro", having a file intro.css in the slides/ folder will make sure it gets loaded along.

Navigating

You can navigate forward by pressing the space key, or the keyboard right arrow. Left arrow navigates to the previous slide.

Transitions

Transitions kick in as you switch from one slide to another. Available transitions are:

fade
Fades slides in/out.
slide
Slides a slide to the left, next one comes from the right.
slice
Slices the slide out. This neat effect doesn't work too well with Internet Explorer, but it degrades nicely so it'll look like no transition effect is being used.

If none is specified, slides will switch instantly.

Plugins

Plugins enhance Shining's default functionalities. They consist simply of a Javascript file that gets loaded according to the name specified in a presentation's configuration file (e.g.: resize loads lib/plugins/resize.js).

Currently available plugins are:

resize
automatic font sizing according to available screen dimensions.
syntaxhighlighter
Adds SyntaxHighlighter support. Refer to documentation available on the original website for examples.
iphone
Makes your presentation look nice, and behave well when viewed on an iPhone.

Though you can write the scripts in any way you want, Shining itself publishes certain events, as in Javascript events, which exist among other reasons for plugins to be able to interfere with the natural order of things without needing to intertwine with Shining's code directly.

Currently the following events are available:

next
as the user requests the next slide.
previous
as the user requests the previous slide.
initialize
when the presentation starts.
slideplay
when a slide plays, after the contents of the slide are loaded onto the stage.

Deploying to Heroku

Deploying a Shining presentation to Heroku allows you to quickly host a presentation online. This can be handy if for any reason you don't have a web server handy to host it.

In a presentation's directory, run:

$ shine deploy

Shining will create a URL friendly name based on the name of your presentation. You can override that by passing an argument to be used as the app name, as in:

$ shine deploy "peanut butter jelly time"

Check the console output for the URL where your presentation is hosted.
